# ADR 0001: DirectoryCore использует проектные адаптеры

Дата: 2026-07-19
Статус: принято

## Контекст

Directory Platform должна обслуживать несколько тематических каталогов, не связывая универсальную логику списка с моделью данных одного проекта.

## Решение

Универсальная модель и таблица `Entry` не создаются. `Facility` остаётся Eloquent-моделью проекта PflegeIndex. `DirectoryCore` определяет нейтральные contracts, value objects и read models, включая `EntryRepository` и `EntrySummary`. Каждый проект предоставляет собственный адаптер репозитория, который преобразует проектные модели в объекты `DirectoryCore`.

Разрешённое направление зависимости:

```text
Project adapter → DirectoryCore
```

`DirectoryCore` не зависит от Eloquent, Laravel paginator или классов конкретного проекта.

## Последствия

Новые каталоги смогут использовать собственные модели хранения без общей ORM-модели. Выбор конкретного адаптера должен выполняться на уровне проекта или composition root; глобальный неоднозначный binding для всех каталогов не вводится.
