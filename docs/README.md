# Directory Platform Documentation

Добро пожаловать в документацию Directory Platform.

Данная документация описывает архитектуру, принципы разработки и долгосрочное развитие платформы.

Directory Platform создаётся как универсальное решение для запуска профессиональных каталогов на единой архитектуре.

Первым проектом платформы является **PflegeIndex**.

# Архитектурная философия

Мы создаём не один каталог.

Мы создаём платформу, на которой можно запускать множество каталогов.

Каждое архитектурное решение должно усиливать платформу в целом.

---

# Основные принципы

- Архитектура важнее скорости.
- Концепция важнее кода.
- Простота лучше сложности.
- Повторное использование лучше копирования.
- Документация является частью продукта.
- Любое изменение должно сохранять целостность платформы.

# Принятие архитектурных решений

Все значимые архитектурные решения документируются.

Для этого используется каталог:

```
docs/decisions/
```

Каждое решение должно содержать:

- описание проблемы;
- рассмотренные варианты;
- принятое решение;
- последствия.

# Последнее правило

Если возникает вопрос:

> "Куда должна относиться новая функциональность?"

нужно задать себе один вопрос:

> **Можно ли использовать её в следующем каталоге?**

Если ответ **да** — это часть Platform Core.

Если ответ **нет** — это часть Project Module.
