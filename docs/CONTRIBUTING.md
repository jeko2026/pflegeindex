# Contributing Guide

Добро пожаловать в Directory Platform.

Этот документ определяет правила участия в разработке проекта.

---

# Основной принцип

Мы создаём не отдельный каталог.

Мы создаём платформу.

Любое изменение должно делать платформу лучше.

---

# Перед началом работы

Перед реализацией новой функциональности необходимо:

- ознакомиться с документацией;
- изучить архитектуру;
- понять границы модулей;
- убедиться, что решение соответствует принципам платформы.

# Архитектурные изменения

Изменения архитектуры не выполняются без предварительного обсуждения.

При необходимости создаётся новое решение в каталоге:

docs/decisions/

---

# Стандарты разработки

Предпочтение отдаётся:

- простому коду;
- читаемым именам;
- повторному использованию;
- минимальному техническому долгу.

---

# Работа с Git

Каждое логически завершённое изменение оформляется отдельным commit.

Сообщения commit должны быть краткими и понятными.

Примеры:

feat:

fix:

docs:

refactor:

test:

---

# Тестирование

Перед завершением задачи необходимо убедиться, что существующие тесты проходят успешно.

Новая функциональность должна сопровождаться тестами, если это оправдано.

---

# Документация

Изменения архитектуры должны сопровождаться обновлением документации.

Документация является частью проекта.

---

# Главный вопрос

Перед написанием нового кода необходимо задать себе вопрос:

Можно ли использовать это решение в следующем каталоге?

Если ответ "да", оно должно стать частью Platform Core.
