# Production Readiness Improvements

## Scope

This release contains only the technical production-readiness corrections identified during the Sprint 4.3 audit. It adds no product features and does not change routes, public URLs, database structure, GeoCore, DirectoryCore, Trust Layer, Content Layer or SEO metadata.

## Repository improvements

- Public CSS and JavaScript use the shared release version `20260722-1`. The versioned URLs allow a one-year immutable cache policy without serving stale code after a release.
- Apache webroots set `Cache-Control: public, max-age=31536000, immutable` for CSS and JavaScript.
- Apache webroots set a 30-day public cache for SVG and PNG assets, whose stable filenames may be replaced independently of templates.
- `deployment/nginx-static-assets.conf` provides the equivalent nginx location blocks for inclusion in the canonical HTTPS vhost.
- The footer logo declares its intrinsic `476 × 104` dimensions to reserve layout space before the SVG loads.
- A global `:focus-visible` outline covers header, footer, navigation, contact and related-facility links as well as buttons and form controls.
- The facility type badge now has a calculated contrast ratio of approximately `5.63:1` against its existing background, exceeding the WCAG AA requirement for normal text.

## Open Graph asset

The shared Open Graph image remains:

```text
https://pflegeindex.com/assets/og-image.png
```

The repository file is `public/assets/og-image.png`. It is a 1254 × 1254 PNG and is required and copied by `scripts/build-production-package.ps1`. Existing tests verify its format, dimensions and metadata references.

The production 404 is a deployment mismatch, not a missing repository asset. Upload the complete public-webroot package, including `assets/og-image.png`; do not upload only Blade or CSS changes.

## Canonical redirects

Application middleware, the Apache webroot rules and `deployment/nginx-canonical-redirect.conf` already define one direct permanent redirect to the exact canonical origin:

```text
https://pflegeindex.com
```

The current live two-hop redirect and explicit `:443` are generated by the hosting nginx configuration before Laravel runs. Apply `deployment/nginx-canonical-redirect.conf` in the hosting panel or ask the provider to apply the equivalent vhosts. Laravel cannot remove an upstream redirect.

## Required deployment actions

1. Build the package from the committed release and verify its manifest and checksums.
2. Upload the complete private core and public webroot according to the existing deployment runbook.
3. Include `deployment/nginx-static-assets.conf` inside the canonical HTTPS nginx server block.
4. Apply `deployment/nginx-canonical-redirect.conf` to the HTTP and HTTPS `www` vhosts.
5. Clear Laravel configuration, route and view caches using the approved deployment procedure.
6. Confirm the deployed CSS and JavaScript URLs contain `v=20260722-1`.

Record the deployed commit hash before declaring production ready. If Trust Layer, Content Layer or the Open Graph image is absent, the public webroot and private core are from different releases and the deployment must not be approved.
